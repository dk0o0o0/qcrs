	
	先有本地库，后有远程库，再关联远程库，使用push推送到远程库，
	每次更改是可从远程库更新到本地库
1.生成公钥   ssh-keygen -t rsa -C 454146035@qq.com  
	     邮箱改成自己的邮箱。设置用户名，密码。
	     可使用命令ssh -T git@github.com  测试能否链接
2.将生成的公钥添加到远程仓库的ssh公钥
3.设置本地仓库，git init
4.关联远程库，$ git remote add qcrs git@github.com:dk0o0o0/qcrs.git

克隆远程仓库：
	$ git clone git@github.com:dk0o0o0/qcrs.git

推送文件到远程仓库：
	1、创建本地分支 dev 命令Git branch dev 
	   切换到分支dev 命令git checkout dev
	  （或者直接使用 创建本地分支dev 并切换到dev分支 
		命令 git checkout -b dev）
	2、推送本地分支dev到远程分支 dev并建立关联关系
	      a.远程已有dev分支并且已经关联本地分支dev且本地已经切换到dev
		  git push qcrs dev（可配置git 命令$ git config --global push.default simple
		  使命令简化为git push)
	      b.远程已有dev分支但未关联本地分支dev且本地已经切换到dev
		 git push -u qcrs/dev
	      c.远程没有有dev分支并，本地已经切换到dev
		git push qcrs dev:dev
	5、删除本地分支dev
	      git branch -d dev
	6、删除远程分支dev
	     git push qcrs  :dev
	     git branch -m | -M oldbranch newbranch 重命名分支，如果newbranch名字分支已经存在，则需要使用-M强制重命名，否则，使用-m进行重命名。
	     git branch -d | -D branchname 删除branchname分支
	     git branch -d -r branchname 删除远程branchname分支
	7、查看本地分支
	      git branch
	8、查看远程和本地分支
	      git branch -a
	
更新远程仓库：
	1.查看远程分支
		$ git remote -v
	2.从远程获取最新版本到本地
		$ git fetch qcrs master:temp
	3.比较本地仓库与下载的temp分支
		$ git diff temp
	4.合并temp分支到本地的master分支
		$ git merge temp
	5.删除temp分支
		$ git branch -d temp
